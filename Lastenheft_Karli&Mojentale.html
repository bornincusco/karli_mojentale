1. Visionen und Ziele

  • /LV10/ Nutzern der Ganttproject-Software soll es ermöglicht werden, die benötigte Zeit für Tasks abhängig von den vorhandenen Ressourcen zu definieren.
  • /LZ10/ Beim Erstellen eines Tasks hat der Nutzer die Option den Task von den vorhandenen Ressourcen abhängig zu machen. Die für einen Task benötigte Zeit soll sich anpassen wenn sich die dafür verantwortlichen Ressourcen verändern.

2. Rahmenbedingungen

Bechreibung der Anwendungsbereiche und Zielgruppen

  • /LR10/ Ganttproject ist eine Open-Source Software die das Zeitmanagement von Projekten erleichtern soll.
  • /LR20/ Die geplante, zusätzliche Funktionalität richtet sich an Nutzer, die sich eine funktionale Verknüpfung zwischen Ressourcen und Tasks wünschen. Dies ist beispielsweise interessant für Projekte bei welchen sich die Ressourcen häufig ändern oder die Zeitabschätzung in Arbeitsstunden einfacher ist als in absoluten Zeitabschnitten.

3. Kontext und Überblick

Festlegung der relevanten Systemumgebung (Kontext) und Überblick über das System

  • /LK10/ Das Effort Driven Scheduling fügt sich in den Bestehenden Rahmen des Task-Systems ein. Es wird lediglich eine weitere Option beim Erstellen bzw. Bearbeiten eines Tasks hinzugefügt.

4. Funktionale Anforderungen

Falls natürlichsprachliche Anforderungen verwendet werden, sollten Sprachtemplates verwendet werden. gemacht werden.

  • /LF10/ Das Feature muss es Nutzern erlauben ein Taskaufwand als fixen Zeitraum oder ressourcenabhängig zu definieren.
  • /LF20/ Das Feature muss es erlauben einem ressourcenabhängigen Task den Aufwand in Arbeitsstunden zuzuordnen.
  • /LF30/ Das Feature muss es erlauben einer Ressource ein tägliches Arbeitspensum zuzuordnen.
  • /LF40/ Das Feature muss die Dauer eines ressourcenabhängigen Tasks anhand der zugeteilten Ressourcen, deren Pensum und deren prozentualen Zeitaufwand berechnen und auf der Übersichtstabelle darstellen können.
  • /LF50/ Das Feature soll die Ferientage der Ressourcen berücksichtigen.
  • /LF60/ Das Feature soll den Nutzer bei Allokation von Ressourcen über 100% warnen.
  • /LF70/ Das Feature soll Tasks mit Abhängigkeiten bei Änderungen dynamisch anpassen. Das heisst Start- und Enddaten so verändern, dass die chronologische Logik bestehen bleibt.
